win7上'gulp'不是内部或外部命令，也不是可运行的程序或批处理文件

By willhu

11月 20 2014 更新日期:11月 21 2014

文章目录

前端时间在折腾gulp前端自动化，本来都是好好地。有一天手欠重装了一次系统，结果PC上的gulp就挂掉了。

输入gulp -v显示：'gulp'不是内部或外部命令，也不是可运行的程序或批处理文件’;
标准的处理办法就是再次装一下gulp
执行：
npm install gulp -g
npm install gulp --save-dev

显示安装成功，理论上应该可以了吧。

呵呵，如果这么简单本文就没有存在的意义了。

安装成功后输入gulp -v，还是显示 'gulp'不是内部或外部命令，也不是可运行的程序或批处理文件’

这是怎么回事？想了一下。应该是node的环境变量问题。于是右键我的电脑->属性->高级系统设置->环境变量

在用户变量里新建变量

PATH: %USERPROFILE%\AppData\Roaming\npm

修改环境变量

在Path中记得加入nodejs

完成后重新打开CMD,输入 gulp -v,显示如下

可以看到全局和当前项目的版本号都读取出来了，说明大功告成。

TIPS: 基于node的很多工具都需要先把node加入系统变量中，所以如果别的工具出现类似问题也是可以依照这样解决的
