HTML5—新增的HTML结构化标记

By willhu

8月 5 2013 更新日期:11月 19 2014

文章目录

相信很多小伙伴们以前习惯以这样的格式来给页面做基础的结构

从而生成一个这样的结构

在HTML4时代，这样的布局可以说毫无问题，甚至可以作为标准。即使在HTML5世界，也能够很好的工作。

这样的结构在人类的眼里可谓是一目了然，但是对于浏览器来说，可以说是“知其然，不知其所以然”。浏览器只能根据class和id的样式进行布局和渲染，初次之外不能活动更多的信息。因为所有的标签都是div，浏览器无法区分中其中的不同，同时搜索引擎在爬取页面的时候，也无法快速的找到需要的信息。

好在HTML5给我们带来了新的元素，来明确的表示页面中的各个部分。所以我们的页面可以改成这样：

代码精简了很多，结构却无任何改变

可惜的是IE家族只有IE9+支持这些新的标签，不过还好，我们有解决办法。

只需要在head部分添加一段js即可

<!—[if lt IE 9]>
<script type=”text/javascript”>
var e=(“abbr,article,aside,audio,canvas,datalist,details,”+ “figure,footer,header,hgroup,mark,menu,meter,nav,output,”+ “progress,section,time,video”).split(‘,’);
for(var i=0;i<e .length;i++){
document.createElement(e[i]);
}
</script>
< ![endif]—>

这样就能解决问题了。

最后肯定有人要问，为什么新增了这些标记呢？幸好我做了准备。以下是截取了opera MAMA爬虫 随机查看了 1806424 个URL中的 5608609 个属性的id所得出的列表，id的命名和使用情况一目了然。

这里只是列出了前20个，如果想了解更多，传送门在此 —> 我是传送门

虽然HTML5在PC端上还无法做到完全替换html4,绝大部分还是需要原始的html4标签来完成页面。但是互联网也不断发展，现代浏览器不断的在更新，增加新功能。IE现在也已经升级到IE11了，IE8以及以下的浏览器早晚都会被这些取代。另外现在移动互联网也在迅速的发展，好在移动端的浏览器绝大部分都能支持这些HTML5标记，所以在移动端上大家可以大胆的使用这些新标记。
