如何给wordpress添加google analytics代码

By willhu

8月 16 2013 更新日期:11月 19 2014

文章目录

博客开通有一段时间了，也写了几篇文章，有没有人看不知道，只是知道每天不少垃圾评论。装了Akismet插件来阻挡垃圾评论，结果在注册的时候一直提示：

Sorry, but this user-agent is banned from using the password retrieval tool.

被墙了，换了两次代理也不行，于是就放弃了。等什么时候找个好点的代理试试。

上面是废话，下面是正题。

说道不知道网站现在访问的如何，所以想起来我可以通过GA来看看网站的情况。光说不做怎么行，立马搞起。

1.首先你要有个google帐户(没有?你是搞IT的嘛)
2.注册 google analytics 帐户
3. 会如下图，要你填写一堆信息，按需填写

4.点击提交后会生成 跟踪 ID 和 跟踪代码 ID就是 UA-XXXXXXXX-X 这样的

5. 然后将GA代码添加到你的网站里就可以了，添加后状态会变成：已安装跟踪代码，这样GA就部署好了。剩下就是细分了，那就是个长久的工程了。

 

其实到这里还没有完，我这个人比较懒，一个一个页面添加ga太麻烦了，不符合我能省事则省事的风格。介于wordpress有着庞大的插件们，于是我百度了一下，发现有不少人推荐 Google Analytics for WordPress** **这个插件。于是研究了一下。安装很简单，从插件库里面搜一下就找到了，只不过都是英文的。

大致的看了一下

第一行的 Manually enter your UA code 就是要填写一开始的那个 UA-XXXXXXXX-X

第二个选项 Where should the tracking code be placed: 是选择GA的位置 默认放在头部 或者 手动插入。 像我这种懒人，先默认放在头部吧

第三个 选项是 Track outbound clicks & downloads: 跟踪跳出点击和下载事件，如选中则会有以下附加选项，貌似是设置将自定义目录下的链接都作为跳出链接跟踪


第四个是高级选项，选中后可以设置一些的自定义的变量属性，选项很多，分得很细，需要仔细看看如何使用。


然后是debug 模式，允许管理员 在ff chrome safari 浏览器上通过监控ga的运行情况



我先用了默认的选项，以后在慢慢折腾。


保存后去GA看下，发现有数据了，说明部署成功了，剩下的就是慢慢的维护了。
